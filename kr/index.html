가격관리 어드민 정식 런칭 및 운영

2025년 구축한 가격관리 백오피스를 정식 런칭. 영업본부 약 150명이 일상 업무 도구로 사용 중이며, 상품·채널별 가격 통계 조회와 경쟁사 대비 가격 우위 판정을 한 화면에서 제공.

대용량 CSV 다운로드는 StreamingResponseBody와 MongoDB cursor 스트리밍을 조합해, 전체 데이터를 메모리에 올리지 않고 row 단위로 흘려보내는 구조로 구현. 수십만 행을 내려받아도 서버 메모리 사용량이 일정하게 유지됨. UTF-8 BOM 삽입으로 엑셀 한글 호환을 처리하고, 필요한 필드만 projection으로 추려 쿼리 부하를 절감.

프론트엔드는 프레임워크 없이 ES Module 단위로 페이지(요약/상품)·공용 모듈을 분리하고, 스냅샷 기반 상태 모듈로 필터·선택·패널 상태를 관리. fetch ReadableStream으로 청크 단위 수신량을 읽어 다운로드 진행률 status bar를 표시하고, 현재 필터/전체 범위를 선택하는 scope 메뉴를 제공. 사용자별 컬럼 레이아웃과 페이지 크기는 localStorage에 영속화하고, 컬럼 추가 시에도 기존 사용자 배치가 깨지지 않도록 merge 로직을 설계.

백엔드는 운영 조회·저장(MongoDB)과 파이프라인 산출물 조회(Databricks JDBC)를 분리한 이원 데이터 소스 구조. 자격증명은 Vault로 관리하고, MTD 추세는 일·시간 버킷 단위로 서버에서 합산하는 집계기를 구현. 도메인 정책·결정 기록을 문서로 남겨 AI 에이전트가 이어서 작업할 수 있는 코드베이스로 운영.

기존에는 개발자가 서버에 접속해 수동으로 Batch 코드를 실행하고 로그를 확인해야 했던 비효율적인 프로세스를 개선. Jenkins Pipeline을 활용해 CI/CD 통합 및 Job 간 의존성 관리 구현.

다중 Job 실행 시 순서 보장과 실패 시 재실행 로직 추가가 어려웠던 점을 해결하기 위해 Parameterized Trigger와 Groovy 스크립트를 사용해 동적 조정 가능하도록 설계.

전체 판매 상품데이터 동기화 작업 Batch 대신 Stream으로 전환

다이나믹 프라이싱을 위한 상품 가격 정보 DB 동기화 프로세스 개선. 기존에는 DB 복제 후 Batch로 데이터를 읽어들이는 방식으로 400GB 규모 데이터를 처리.

Batch가 Truncate 후 가격 정보를 다시 채우는 방식이라 동기화 중 다이나믹 프라이싱 잡 실행 시 데이터 공백으로 오류 발생. 이를 해결하기 위해 Truncate 대신 임시 테이블에 데이터를 쌓은 후 RENAME으로 테이블을 바꿔치기하는 전략 도입.

상품 수 증가로 동기화 잡 시간이 길어지며 잡 간 겹침 발생. 최종적으로 mysqldump로 초기 적재 후, Kafka 컨슈밍 앱을 개발해 변동분만 실시간 반영하도록 전환.

할인 반영 상품 수를 5천만 개에서 2억 8천만 개로 확대, 가격 데이터 실시간화 달성. Poison Pill 처리 로직과 Row Lock 방지 트랜잭션 설계로 안정성 강화.

상품 가격 변동 시스템 메인 Batch 리뉴얼

Kafka 기반 병렬 처리와 Producer-Consumer 패턴을 적용하여 가격 변동 처리 속도를 최적화.

Enum과 Map을 통한 분기처리와 병렬처리를 활용해 프로세스 시간을 1.5시간에서 10~20분으로 단축.

사내 기술 발표 (4회)

롯데e커머스에서의 기술블로그 기반 온라인 발표(2022)를 포함해 총 4회 진행. G마켓 전사 Staff Learning Session(SLS)에서는 세 차례 Speaker로 참여.

첫 번째 SLS 발표에서는 '누구나 돈 한 푼 안 들이고 AI의 도움으로 업무를 대신할 프로그램 개발하기'를 주제로, AI 무료 활용법, 서버 없이 프로그램 실행 방법, 무료 코드 에디터 사용법을 소개하고 Grok을 활용한 Chrome Extension 시연을 진행.

두 번째 발표에서는 정적·동적 크롤링 기술과 AI의 6단계 발전(인지형, 생성형, RAG, MCP, Agent, 물리형)을 주제로, 최신 MCP(Model Context Protocol) 기술을 접목한 AI Browser Agent를 직접 시연하며 차세대 웹 자동화 및 업무 효율화 방안을 공유.

세 번째 발표(2026)에서는 'AI와 비즈니스의 상관관계'를 주제로, 클라이언트-서버-DB 웹 구조에서 출발해 AI 모델·웹 구독·API 과금이 어떻게 다른지, AI 에이전트가 등장한 배경과 안전한 운영 방식(클라우드 VM·컨테이너·전용 머신 분리), 모델 라우팅과 Heartbeat로 인한 비용 누수 같은 비용 최적화 노하우를 비개발자 눈높이로 설명. '고객이 에이전트가 되는 시대'에 API·CLI 통로가 없는 서비스가 겪을 불리함과 광고·가격비교·CX 등 이커머스 비즈니스 영향까지 고찰을 공유하고, AI 에이전트로 만든 조종 앱을 라이브 시연.
